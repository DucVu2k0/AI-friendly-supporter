# ✅ Definition of Done

## Một task được coi là HOÀN THÀNH khi:

### 📋 Content Quality
- [ ] Trả lời đúng & đủ câu hỏi của user
- [ ] Không có thông tin sai lệch
- [ ] Không có noise/thông tin thừa

### 🔍 Citations & Sources
- [ ] Mỗi khái niệm/công thức có ghi: `(SGK [Môn] [Lớp] [Tập], trang X)`
- [ ] Trang số chính xác hoặc `(trang không xác định)` nếu không tìm được
- [ ] Đường dẫn file gốc rõ ràng: `09_Teacher_Assistant/Tai_Lieu/...`

### 📐 Format & Structure
- [ ] File markdown đúng syntax
- [ ] Tuân theo cấu trúc quy định (nếu có)
- [ ] Tiêu đề & heading rõ ràng

### 📏 File Size
- [ ] ≤ 200 dòng (hoặc >= nếu do user yêu cầu)
- [ ] Không thừa không thiếu

### 🛠️ Output Files
- [ ] File được lưu đúng vị trí: `Skill/`, `Instruction/`
- [ ] Tên file đúng naming convention
- [ ] Commit message rõ ràng (nếu commit code)

### ✨ User Satisfaction
- [ ] Output trực tiếp trả lời câu hỏi
- [ ] Không cần giải thích thêm
- [ ] User không cần hỏi lại: "bạn lấy từ đâu?"

---

## ❌ Dấu hiệu CHƯA XONG:

- ❌ User hỏi lại câu hỏi tương tự
- ❌ Output có thêm thống tin user không yêu cầu
- ❌ Không biết source từ đâu
- ❌ File định dạng lộn xộn
- ❌ Thông tin mâu thuẫn giữa các phần

---

## Áp dụng cho dự án này:

| Task | Done khi |
|------|---------|
| Tạo file Skill Toán Lớp 7 | ✅ File có định nghĩa, công thức, ví dụ + cite trang SGK |

**Principle:** Hỏi gì, trả lời đó. Không thêm không bớt.
